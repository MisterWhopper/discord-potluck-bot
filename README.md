# discord-potluck-bot
This project is DOA! Don't try to do this

## Why this was abandoned
To quote the final (non-trivial) commit on this repo:

> So, after experimenting with this for a handful of days, I have come to
the conclusion that doing this kind of task via Discord bot is just not
UX-friendly enough to be viable.
> 
> The main issue is modals; while these are awesome for prompting users
for info, they can only really be created if the user runs a command.
Most users aren't likely to be interested enough in a potluck to run a
command, even if the bot tells them exactly what to type.
> 
> The real nail in the coffin though is that modals cannot have more than
5 UI componenets, nor can a modal be chained into another one. This
makes complex UI forms impossible, as you can't even have a "second
page" modal after the user completes the first one.
> 
> Things that should be webforms should maybe just be webforms.
